# 🎲 Кубики

Покрокова веб-гра в кубики на **1–8 гравців**: школи, комбінації та автоматичний підрахунок результатів. Кубики кидаються в реальному житті, а застосунок — це електронний бланк, який веде черговість ходів і рахунок.

Працює **повністю офлайн** (чистий HTML/CSS/JS, без жодних залежностей) і встановлюється на телефон як **PWA**.

## Як грати

1. Оберіть кількість гравців і введіть імена.
2. Гравці ходять по черзі. За хід заповнюється рівно одна вільна клітинка.
3. **Школи (1–6).** Бали = (кількість кубиків з потрібним числом − 3) × номінал. Три кубики — це нуль. Якщо в кінці гри сума по школах від'ємна, до неї додається штраф −50.
4. **Комбінації.** Зібрали з першого кидка — закриваєте ×2. З другого чи третього — звичайні бали. Не вдалося — викреслюєте (0).

| Комбінація | Бали |
|---|---|
| Пара | 5 |
| Дві пари | 10 |
| Трикутник | 15 |
| Малий стріт | 20 |
| Великий стріт | 30 |
| Фулхаус | 35 |
| Каре | 40 |
| Покер | 50 |
| Шанс | сума граней (5–30) |

Коли всі заповнять усі 15 клітинок — застосунок показує переможця.

## Запуск локально

Просто відкрийте `index.html` у браузері. Для перевірки PWA / service worker запустіть локальний сервер (service worker не працює з `file://`):

```bash
npx serve .
# або
python3 -m http.server 8000
```

## Деплой

Це статичний сайт — збірка не потрібна.

### Vercel
1. Залийте репозиторій на GitHub.
2. На vercel.com → **Add New → Project** → імпортуйте репозиторій.
3. Framework Preset: **Other**, Build Command: *порожньо*, Output Directory: `.`
4. **Deploy**. Отримаєте https-посилання з автодеплоєм на кожен push.

### Альтернативи
- **GitHub Pages** — Settings → Pages → Branch `main` / root.
- **Netlify** — перетягніть папку на app.netlify.com/drop або підключіть репозиторій.
- **Cloudflare Pages** — підключіть репозиторій, build command порожній.

```
├─ sw.js                 # service worker (офлайн-кеш)
├─ icons/
│  ├─ icon-192.png
│  └─ icon-512.png
├─ README.md
├─ .gitignore
└─ LICENSE
```

## Технології

Навмисно без фреймворків: ванільний JavaScript і CSS. Гра суто клієнтська, без бекенду, тож статичний файл — найпростіший і найнадійніший варіант. Жодного кроку збірки, нічого ламати.
