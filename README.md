# 🎲 Кубики

Покрокова веб-гра в кубики на **1–8 гравців**: школи, комбінації та автоматичний підрахунок результатів. Кубики кидаються в реальному житті, а застосунок — це електронний бланк, який веде черговість ходів і рахунок.

Працює **повністю офлайн** (чистий HTML/CSS/JS, без жодних залежностей) і встановлюється на телефон як **PWA**.

## Як грати

1. Оберіть кількість гравців і введіть імена.
2. Гравці ходять по черзі. За хід заповнюється рівно одна вільна клітинка.
3. **Грані (1–6).** Бали = (кількість кубиків з потрібним числом − 3) × номінал. Три кубики — це нуль. Якщо в кінці гри сума по гранях від'ємна, до неї додається штраф −50.
4. **Комбінації.** Зібрали з першого кидка — закриваєте ×2. З другого чи третього — звичайні бали. Не вдалося — викреслюєте (0).

| Комбінація | Бали |
|---|---|
| Пара | 5 |
| Дві пари | 10 |
| Трикутник | 15 |
| Малий ряд | 20 |
| Великий ряд | 30 |
| Фулхаус | 35 |
| Каре | 40 |
| Покер | 50 |
| Бонус | сума граней (5–30) |

Коли всі заповнять усі 15 клітинок — застосунок показує переможця.

## Тестування локально

Відкрийте `index.html` у браузері. Якщо треба перевірити PWA / service worker (вони не працюють з `file://`):

```bash
npx serve .
# або
python3 -m http.server 8000
```

## Деплой

Сайт розгорнуто через **GitHub Pages**. Щоб запропонувати зміни — зробіть pull request у гілку `main`. Після перевірки та мерджу сайт оновиться автоматично.

## Структура

```
kubiky/
├─ index.html            # вся гра (HTML + CSS + JS в одному файлі)
├─ manifest.webmanifest  # PWA-маніфест (встановлення на телефон)
├─ sw.js                 # service worker (офлайн-кеш)
├─ icons/
│  ├─ icon-192.png
│  └─ icon-512.png
├─ README.md
├─ .gitignore
└─ LICENSE
```

## Технології

Навмисно без фреймворків: ванільний JavaScript і CSS. Гра суто клієнтська, без бекенду, тож статичний файл — найпростіший і найнадійніший варіант. Жодного кроку збірки, нічого ламати.
